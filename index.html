What I'm Working On

  • Disaggregated Resources and Cloud Control Planes

    A trend in cloud data centers is disaggregating resources such as memory and storage. "Disaggregating" means placing the resources in shared pools from which they can be allocated to any physical server, in contrast to the traditional approach where each server has a fixed amount of dedicated resources. The benefit is better utilization of resources, which translates to lower costs. But this paradigm shift requires rethinking all cloud orchestration. For example: how should cluster scheduling (i.e. assigning jobs to servers) be done in the light of disaggregation? Probably not in the same way as before. This is one example of the problems I'm working on in this space, together with Prof. Ada Gavrilovska.

  • Performance Model of Microservices

    "Microservices" is an architectural pattern where complex applications are organized as a distributed system of small, independently developed and deployed services. While they greatly increase development speed and organizational agility, their highly distributed nature can worsen their performance, and make it harder to understand how to allocate resources between all the services to reach the SLOs. I'm working on using queuing theory to model microservices to predict their performance and drive resources allocation on the basis of the predictions, together with Prof. Calton Pu.
